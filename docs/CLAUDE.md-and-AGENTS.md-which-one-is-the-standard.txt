CLAUDE.md and AGENTS.md ? which one is the standard?

Identified configuration file standards for AI coding agents

The user is asking about CLAUDE.md vs AGENTS.md - which is the standard configuration file for AI coding agents.

Both are real, but they serve slightly different purposes:

CLAUDE.md is Anthropic's standard — it's the configuration file that Claude Code reads automatically when you open a project. It tells Claude about the codebase, conventions, commands to run, and anything else you want it to know. Anthropic defined and documented this format.

AGENTS.md emerged more as a community/multi-agent convention — the idea being a more generic, tool-agnostic name that any AI coding agent (not just Claude) could read. It gained traction as people wanted a single file that works across Claude, Cursor, Copilot, etc.

In practice:

• Use CLAUDE.md if you're primarily using Claude Code — it's the officially supported format

• Use AGENTS.md if you want something more tool-agnostic or are working in a multi-agent setup

• Some projects include both, or just pick one and commit

Claude Code will read CLAUDE.md by default. There's no single industry-wide standard yet — it's still a bit of a wild west as AI coding tools evolve. OpenAI's Codex also recently started supporting AGENTS.md, which has given that name more momentum as a cross-tool convention.

Short answer: CLAUDE.md is Anthropic's official standard for Claude Code. AGENTS.md is the emerging community convention for tool-agnostic agent instructions.

If AGENTS.md is your primary file:

This way you only maintain one file but both names resolve to it. Works great if you want cross-tool compatibility without duplicating content.

.cursorrules Status in 2026: Officially Deprecated (But Not Dead)

Cursor's own docs now say .cursorrules is "still supported but will be deprecated" and explicitly recommend migrating to Project Rules for more control, flexibility, and visibility. 
InfoWorld

Cursor's official recommendation as of now is to migrate to either Project Rules (.cursor/rules/ as .mdc files) or AGENTS.md. 
Medium

So .cursorrules is in a zombie state — works, but officially on its way out.

The Cursor File Hierarchy (2026)

There are now three distinct layers in Cursor:

1. .cursor/rules/*.mdc — Project Rules (the new native format) Project rules are stored in .cursor/rules/ as .mdc files, version-controlled per project, and evaluated in this priority order: Local (manual) → Auto Attached → Agent Requested → Always Apply. Legacy .cursorrules is still supported but deprecated. 
SmartScope

2. AGENTS.md — Cross-tool standard AGENTS.md is a plain markdown file for defining agent instructions — supported in the project root and subdirectories. Unlike Project Rules, it has no metadata or complex configurations, making it perfect for simple, readable instructions without overhead. 
Medium

3. .cursorrules — Legacy, deprecated Still read, but Cursor won't generate new ones. There's even a bug report where Cursor's own /Generate Cursor Rules command accidentally created a .cursorrules file instead of the new .mdc format — confirming the old format is being phased out even internally. 
GitHub

.cursorrules vs SKILLS.md — How Do They Compare?

.cursorrules | .cursor/rules/*.mdc | AGENTS.md | SKILLS.md
Status | Deprecated	| Active native	| Active standard |	Active
Scope | Always-on, whole project | Scoped, conditional | Simple always-on | Loaded on-demand
Format | Plain markdown | MDC with metadata | Plain markdown | Plain markdown

The key distinction between rules and skills is this: rules with alwaysApply: true are great for coding style, error handling, and framework conventions — they load every time. Skills are better for multi-step workflows and deployment procedures because they keep the context window clean by only loading when needed. Rules are not deprecated by skills — they do different things.

The Community Consensus (Feb 2026)

Most coding tools have consolidated around AGENTS.md as the cross-tool standard. The recommended setup is: AGENTS.md in the project root for shared instructions, tool-specific rules files for advanced per-tool configuration, and skills for procedural workflows.

Bottom line: .cursorrules is legacy — migrate away from it. AGENTS.md is the cross-tool winner for simple instructions. .cursor/rules/*.mdc is Cursor's own advanced format for fine-grained control. SKILLS.md complements all of them for on-demand procedural context.
